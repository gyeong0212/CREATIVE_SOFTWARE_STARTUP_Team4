# MAK Link: AI 발권 안내 데모 웹앱 Blueprint

## 1. 프로젝트 개요

"MAK Link"는 고령층 사용자를 위한 AI 기반 철도 승차권 발권 안내 데모 웹 애플리케이션입니다. 복잡한 무인 발권기(키오스크) 앞에서 어르신들이 마치 친절한 안내원과 대화하듯 쉽고 편안하게 승차권을 예매할 수 있도록 돕는 것을 목표로 합니다.

- **타겟 유저**: 디지털 기기 사용에 익숙하지 않은 고령층
- **핵심 기능**: 자연어 기반 대화형 승차권 예매, AI 추천, 간결하고 큰 UI
- **기술 스택**: Next.js (App Router), React (Server Components), Tailwind CSS, Google Gemini API
- **주요 특징**:
    - Gemini API를 활용하여 사용자의 의도를 파악하고, 부족한 정보를 질문하여 예매 과정을 안내합니다.
    - 실제 예매 데이터를 사용하지 않고, 미리 정의된 더미 데이터를 사용하여 데모의 완성도를 높입니다.
    - 고령층 사용자를 위해 직관적인 디자인, 큰 글씨, 명확한 버튼 등 접근성을 최우선으로 고려합니다.

## 2. 디자인 및 스타일 가이드

전체적으로 따뜻하고 친근한 느낌을 주면서도, 정보가 명확하게 전달될 수 있도록 디자인합니다.

- **폰트**: 가독성이 높은 Pretendard 또는 유사한 산세리프 폰트를 사용합니다. 전체적으로 기본 폰트 크기를 크게 설정합니다.
- **색상 팔레트**:
    - **주조색**: 편안하고 신뢰감을 주는 파란색 계열 (`#3B82F6`)
    - **보조색**: 따뜻한 느낌의 주황색 계열 (`#F97316`)을 버튼 등 상호작용 요소에 사용합니다.
    - **배경색**: 눈의 피로를 덜어주는 부드러운 회색 (`#F3F4F6`) 또는 흰색을 사용합니다.
    - **텍스트**: 기본적으로는 짙은 회색 (`#1F2937`)을 사용하되, 강조가 필요한 부분은 주조색을 사용합니다.
- **레이아웃**:
    - 중앙 정렬을 기본으로 하여 사용자의 시선이 자연스럽게 콘텐츠에 집중되도록 합니다.
    - 각 요소 간의 간격(spacing)을 충분히 확보하여 답답하지 않고 시원한 느낌을 줍니다.
    - 모바일 화면에서도 최적의 사용성을 제공하는 반응형 디자인을 적용합니다.
- **컴포넌트 스타일**:
    - **버튼**: 크고 둥근 모서리(rounded corners)를 가지며, 명확한 텍스트 라벨을 포함합니다. 누르는 느낌이 확실히 들도록 그림자 효과와 호버(hover) 효과를 적용합니다.
    - **입력창**: 버튼과 마찬가지로 크고 명확하게 디자인하며, 현재 포커스된 상태를 시각적으로 알려줍니다.
    - **카드**: 승차권 후보를 표시하는 카드는 부드러운 그림자 효과를 주어 입체감을 느끼게 합니다.

## 3. 기능 구현 계획

### 1단계: 프로젝트 초기 설정 (완료)
- `npm install @google/generative-ai`
- `.env.local` 파일 생성 및 `GEMINI_API_KEY` 환경 변수 설정
- `.gitignore` 파일에 `.env.local` 및 `node_modules` 추가

### 2단계: 기본 구조 및 UI/UX 구현
- **파일 구조 정리**: 기존 `js` 파일을 `tsx`로 전환하고, 불필요한 파일을 정리합니다.
- **`globals.css` 수정**: Tailwind CSS 기본 설정 및 전체 폰트, 배경색 등을 정의합니다.
- **Layout (`layout.tsx`)**: 모든 페이지에 공통적으로 적용될 루트 레이아웃을 설정합니다. (헤더, 푸터 등)
- **메인 페이지 (`page.tsx`)**: 초기 화면 UI 구현 (안내 문구, 텍스트 입력창, "어디로 갈까요?" 버튼)
- **컴포넌트 구현**:
    - `Header.tsx`: 진행 단계(예: 1/5)를 표시할 헤더 컴포넌트.
    - `TicketCard.tsx`: 승차권 후보를 표시할 카드 컴포넌트.
- **더미 데이터 생성**: `src/lib/dummyData.ts` 파일에 다양한 시간대의 승차권 후보 더미 데이터를 생성합니다.

### 3단계: AI 연동 및 서버 액션 구현
- **Server Action (`actions.ts`)**:
    - `askToGemini` 함수: 사용자의 입력을 받아 Gemini API에 전달하고, 응답을 받아오는 서버 액션을 정의합니다.
    - API 키는 `process.env.GEMINI_API_KEY`를 통해 안전하게 서버 측에서만 사용합니다.
- **상태 관리**: `useState`와 `useReducer`를 사용하여 대화 흐름(사용자 질문, AI 답변, 다음 질문 등)과 예매 상태를 관리합니다.

### 4단계: 화면 흐름 및 로직 구현
1.  **시작 화면**: 사용자가 목적지를 입력하고 버튼을 누르면 `askToGemini` 서버 액션을 호출합니다.
2.  **AI 질문/답변**:
    - Gemini API의 응답을 분석합니다. 만약 부족한 정보(인원, 시간 등)가 있다면, AI가 생성한 질문을 사용자에게 보여줍니다.
    - 답변은 간단한 버튼 선택 형태로 제시하여 사용자의 편의성을 높입니다.
    - 모든 정보가 수집될 때까지 이 과정을 반복합니다.
3.  **후보 정리 화면**:
    - 모든 정보가 수집되면, 서버 액션은 더미 데이터를 기반으로 "가장 빠른 시간", "가장 저렴한", "여유 있는 시간" 3가지 추천 후보를 생성하여 반환합니다.
    - `TicketCard.tsx` 컴포넌트를 사용하여 3개의 후보를 화면에 렌더링합니다.
4.  **결제 전 확인 화면**:
    - 사용자가 3가지 후보 중 하나를 선택하면, 해당 승차권의 상세 정보를 크게 보여주는 확인 화면으로 전환됩니다.
    - "이 표로 할까요?" 와 같은 명확한 질문과 "예매하기", "다시 고르기" 버튼을 제공합니다.
5.  **완료 화면**:
    - "예매하기" 버튼을 누르면 최종 완료 화면으로 이동합니다.
    - "예매가 완료되었습니다" 메시지와 함께 탑승 위치(예: 3번 승강장) 정보를 안내합니다.

### 5단계: 마무리 및 배포 준비
- **최종 테스트**: 전체 예매 흐름을 처음부터 끝까지 테스트하며 오류나 어색한 부분을 수정합니다.
- **Git 초기화**:
    - `git init`
    - `git add .`
    - `git commit -m "Initial commit: MAK Link project setup"`
- **GitHub Push 안내**: 사용자가 자신의 원격 저장소에 푸시할 수 있도록 명령어 `git remote add origin <YOUR_REPOSITORY_URL>` 및 `git push -u origin main` 을 안내합니다.
